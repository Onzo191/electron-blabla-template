# Markdown UI Templates Specification - AI Agents Module

One of the core features of the AI Agents module is its capability to translate structured JSON payloads embedded inside Markdown responses (generated by AI models) into interactive, rich UI components. This document defines the parser rules, boundary markers, and data schemas for all supported custom template blocks.

---

## 1. Boundary Detection & Parsing Engine

When the bot returns markdown contents (either completed or streaming), the client parses and extracts sections based on pre-compiled regular expressions.

### A. Block Boundary Tokens
Structured template blocks are enclosed between designated opening and ending marker lines:

| Template Block Name | Opening Marker | Ending Marker |
| :--- | :--- | :--- |
| **Media** (Images & Videos) | `====MEDIA====` | `====END_MEDIA====` |
| **Suggestion** (Follow-up prompts) | `====SUGGESTIONS====` | `====END_SUGGESTIONS====` |
| **Support Contact** (IT desk info) | `====SUPPORT_CONTACT====` | `====END_SUPPORT_CONTACT====` |
| **References** (Citations) | `====REFERENCES====` | `====END_REFERENCES====` |
| **Closing** (Final statement) | `====CLOSING====` | `====END_CLOSING====` |
| **Attachments** (Files list) | `====ATTACHMENTS====` | `====END_ATTACHMENTS====` |
| **Export Artifact** (Generated file) | `====EXPORT_ARTIFACT====` | `====END_EXPORT_ARTIFACT====` |
| **Export Format Picker** (Formats list) | `====EXPORT_FORMAT_PICKER====` | `====END_EXPORT_FORMAT_PICKER====` |
| **Colleague Card** (Employee search) | `====ASK_COLLEAGUES_TOOL====` | `====END_ASK_COLLEAGUES_TOOL====` |

### B. Group Parsing Regular Expression
The system evaluates blocks dynamically by grouping keys into a single expression pattern:
```javascript
const PATTERN_REGEX = /(?:====MEDIA====\s*(?<media>[\s\S]*?)\s*====END_MEDIA====)|(?:====SUGGESTIONS====\s*(?<suggestion>[\s\S]*?)\s*====END_SUGGESTIONS====)|.../g;
```

### C. Clean AI Citations & Cleanup
Before compiling the raw response as standard Markdown (`prose`), the client performs sanitation filters:
- `REF_REGEX = /\[ID:([^\]]+)\](,\s*|\.\s*|\s*)/g`: Strips internal citation tags (e.g. `[ID:document.pdf]`) and maps them to clean numbered citations (e.g. `[1]`, `[2]`).
- `FILECITE_REGEX = /\S*(?:filecite|turn\d+file\d+)\S*/gu`: Eliminates dangling keywords like `filecite` and `turn1file2` to maintain clean prose layouts.

## 2. Nine Rich UI Template Widgets Specs

### 1. Colleague Card (Employee directory search)
- **Tag Marker**: `====ASK_COLLEAGUES_TOOL====`
- **Purpose**: Displays profile sheets matching corporate employee lookups.
- **JSON Structure**:
  ```json
  {
    "intro": "Here is the colleague profile I found:",
    "data": {
      "name": "Jane Doe",
      "title": "Software Engineer",
      "department": "Zalo Group",
      "office": "VNG Campus",
      "phone": "+84 909xxxxxx",
      "zalo": "+84 909xxxxxx",
      "email": "janedoe@vng.com.vn",
      "lineManager": "John Smith (johns)",
      "lineManagerTitle": "Technical Lead",
      "avatar": "https://avatar-url.jpg",
      "username": "janedoe",
      "employeeId": "12345",
      "tenure": "3 years"
    }
  }
  ```
- **Interactive Handlers**:
  - Clicking phone/email fields triggers quick copy to clipboard with a success toast.
  - **Line Manager Field**: Clicking the manager's name auto-fills a lookup prompt like `"Look up John Smith"` in the chat box and submits it.
  - **Mail / Zalo Links**: Anchor buttons opening Outlook/Zalo web directly.

---

### 2. Media Grid (Image & Video Gallery)
- **Tag Marker**: `====MEDIA====`
- **Purpose**: Lays out visual image/video portfolios.
- **JSON Structure**:
  ```json
  {
    "intro": "Check out these photos:",
    "items": [
      {
        "url": "https://domain.com/office.png",
        "alt": "Campus workspace",
        "mediaType": "image"
      },
      {
        "url": "https://domain.com/tutorial.mp4",
        "alt": "VPN setup video guide",
        "mediaType": "video"
      }
    ]
  }
  ```
- **Interactive Handlers**:
  - Grid displays up to 4 items. If extra items exist, the last element overlays an overlay tag showing remaining items count (e.g. `+3`).
  - Clicking any item triggers a fullscreen **Lightbox Viewer** with carousel capabilities and HTML5 video play/pause controls.

---

### 3. Suggestion (Suggested follow-up Prompts)
- **Tag Marker**: `====SUGGESTIONS====`
- **Purpose**: Displays quick-click prompt cards guiding user workflows.
- **JSON Structure**:
  ```json
  {
    "intro": "You might also want to ask about:",
    "items": [
      "VNG Campus parking registration guides",
      "Private medical health packages 2024"
    ]
  }
  ```
- **Interactive Handlers**:
  - Renders as custom outline action buttons.
  - Clicking a button injects the string directly into the query input and posts it immediately.
  - Disabled/Hidden when the message is not the final message in the thread to ensure clean conversation history.

---

### 4. Support Contact (Corporate IT Desk Help Cards)
- **Tag Marker**: `====SUPPORT_CONTACT====`
- **Purpose**: Contacts lists for technical/administrative support teams.
- **JSON Structure**:
  ```json
  {
    "intro": "For technical help, reach out to these contacts:",
    "items": [
      {
        "domain": "IT Helpdesk",
        "name": "Helpdesk Campus",
        "avatar": "https://avatar-url",
        "phone": "Ext: 1111",
        "email": "helpdesk@vng.com.vn"
      }
    ]
  }
  ```
- **Interactive Handlers**:
  - Standard card widgets. Phone/email fields support copy actions.

---

### 5. References (Citations & Sources)
- **Tag Marker**: `====REFERENCES====`
- **Purpose**: Document citations list backing bot assertions.
- **JSON Structure**:
  ```json
  {
    "intro": "Sources used to compile this information:",
    "items": [
      {
        "title": "Employee Handbook 2024.pdf",
        "url": "https://internal-drive/handbook.pdf"
      }
    ]
  }
  ```
- **Interactive Handlers**:
  - Displays as a numbered catalog. Clicking titles opens target URL in a new tab.

---

### 6. Attachments (Direct Downloads List)
- **Tag Marker**: `====ATTACHMENTS====`
- **Purpose**: Provides file templates downloadable directly.
- **JSON Structure**:
  ```json
  {
    "intro": "Download registration form spreadsheets:",
    "items": [
      {
        "name": "Device_Request_Form.xlsx",
        "size": "45 KB",
        "url": "https://internal-bapi/downloads/form.xlsx"
      }
    ]
  }
  ```
- **Interactive Handlers**:
  - Clicking rows triggers standard file download events.

---

### 7. Export Artifact (Finished Reports / Assets Cards)
- **Tag Marker**: `====EXPORT_ARTIFACT====`
- **Purpose**: Renders download cards for compilation documents assembled by the agent.
- **JSON Structure**:
  ```json
  {
    "attachmentId": "artifact-uuid-99",
    "fileName": "Q2_Performance_Summary.pdf",
    "format": "pdf",
    "sizeBytes": 2048500,
    "viewUrl": "v2/api/attachments/artifact-uuid-99/view",
    "downloadUrl": "v2/api/attachments/artifact-uuid-99/download"
  }
  ```
- **Interactive Handlers**:
  - Styled as a document sheet widget with extension-based icons (PDF, Word, Excel, PowerPoint).
  - Contains a **View** button (opens viewer tab) and a **Download** action button (triggers a binary buffer request using `aiChatbotServices.getExportArtifactBlob` and writes it as local file storage).

---

### 8. Export Format Picker (File Formats Selector)
- **Tag Marker**: `====EXPORT_FORMAT_PICKER====`
- **Purpose**: Prompts users to select their desired file format for exports.
- **JSON Structure**:
  ```json
  {
    "availableFormats": ["pdf", "docx", "xlsx"]
  }
  ```
- **Interactive Handlers**:
  - Selecting a format (e.g. PDF) triggers an optimistic chat command (e.g. `"Export as PDF"`) and posts it to the agent.

---

### 9. Closing (Bot outro card)
- **Tag Marker**: `====CLOSING====`
- **Purpose**: Houses the bot's closing advice or warnings in a dedicated prose container.
- **Structure**: Either plain text or JSON object `{ "text": "..." }`.
