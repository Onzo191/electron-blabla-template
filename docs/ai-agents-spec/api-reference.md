# API Reference Specification - AI Agents Module

This document defines the complete API specification connecting the React front-end to the backend services of the AI Chatbot/Agents module. It covers the RESTful services for configuration and CRUD management (Agents, Conversations, Attachments, MCP) as well as the Server-Sent Events (SSE) streaming protocol.

---

## 1. General Config & Authentication Headers

### API Base URLs
- **Local Dev**: `http://localhost:3000` (controlled by `LOCAL_DEV` and `API_GATEWAY_URL` in `config.ts`).
- **Production**: Communicates via `API_GATEWAY_URL`, prefixing services with `/v2/api/...` or `/api/v2/...` depending on routing rules.

### Required Headers
All client requests must carry authentication tokens and environment identifiers:
- `Authorization: Bearer <Access_Token>`
- `Content-Type: application/json` or `multipart/form-data` (when uploading file blobs).

## 2. REST Endpoints List

### A. Agents Management Services

#### 1. Retrieve Default Agent
- **Endpoint**: `GET v2/api/agent/end-user/default`
- **Response**:
  ```json
  {
    "data": [
      {
        "id": "agent-uuid-1",
        "name": "VNG Assistant",
        "icon": "icon-url-or-emoji",
        "description": "Supports answering questions on VNG internal policies...",
        "greetingMessages": [
          {
            "greetingMessageTranslations": [
              { "language": "vi", "content": "<p>Xin chào! Tôi có thể giúp gì cho bạn hôm nay?</p>" },
              { "language": "en", "content": "<p>Hello! How can I help you today?</p>" }
            ]
          }
        ]
      }
    ]
  }
  ```

#### 2. Retrieve Available Agents List
- **Endpoint**: `GET v2/api/agent/end-user?page={page}&size={size}`
- **Response**: Paginated list of active agents displayed on the Sidebar.

#### 3. Retrieve Agent Details by ID
- **Endpoint**: `GET v2/api/agent/{agentId}/basic-info`
- **Response**: Active configuration properties for the agent (e.g. prompt Topics, source configs, mcpConfigs).

### B. Conversation Management Services

#### 1. Retrieve Conversation List (Recents)
- **Endpoint**: `GET v2/api/conversations?page={page}&size={size}&search={searchQuery}`
- **Response**:
  ```json
  {
    "data": {
      "items": [
        {
          "id": "conversation-uuid-1",
          "name": "Internal Insurance Policy 2024",
          "createdAt": "2026-07-01T12:00:00Z"
        }
      ],
      "pagination": {
        "page": 1,
        "size": 12,
        "total": 50,
        "hasNextPage": true
      }
    }
  }
  ```

#### 2. Retrieve Pinned Conversations
- **Endpoint**: `GET v2/api/conversations/pinned?page={page}&size={size}`

#### 3. Pin a Conversation
- **Endpoint**: `POST v2/api/conversations/{conversationId}/pin`

#### 4. Unpin a Conversation
- **Endpoint**: `DELETE v2/api/conversations/{conversationId}/pin`

#### 5. Initialize Empty Conversation
- **Endpoint**: `POST v2/api/conversations/init`
- **Payload**: `{ "agentId": "string" }`
- **Response**: Returns metadata including the new conversation's UUID.

#### 6. Retrieve Conversation Message History
- **Endpoint**: `GET v2/api/conversations/{conversationId}?page={page}&size={size}`
- **Response**: Historical array of messages sorted in reverse chronological order (newest first).

#### 7. Rename Conversation
- **Endpoint**: `PATCH v2/api/conversations/{conversationId}`
- **Payload**: `{ "name": "New Conversation Title" }`

#### 8. Delete Conversation
- **Endpoint**: `DELETE v2/api/conversations/{conversationId}`

### C. File Attachments & Artifacts Services

#### 1. Upload Attachment to Conversation
- **Endpoint**: `POST v2/api/conversations/{conversationId}/attachments/upload`
- **Method**: `POST` (Multipart Form Data)
- **Payload**: `file` (Binary File Blob)
- **Response**:
  ```json
  {
    "data": {
      "id": "attachment-uuid-1",
      "name": "invoice.pdf",
      "size": 154200,
      "status": "processing" // processing | success | failed
    }
  }
  ```

#### 2. Retrieve Conversation Attachments List
- **Endpoint**: `GET v2/api/conversations/{conversationId}/attachments?page={page}&size={size}`

#### 3. Delete Attachment File
- **Endpoint**: `DELETE v2/api/attachments/{attachmentId}`

#### 4. Fetch Attachment View & Download Links
- **Endpoint**: `GET v2/api/attachments/{attachmentId}/view-url`
- **Endpoint**: `GET v2/api/attachments/{attachmentId}/download-url`

#### 5. Fetch Generated Artifacts list
- **Endpoint**: `GET v2/api/conversations/{conversationId}/artifacts`
- **Response**: Returns compilation summaries generated by the bot.

### D. System Extensions (Feedback, Export & MCP)

#### 1. Submit Message Feedback
- **Endpoint**: `POST v2/api/feedback`
- **Payload**:
  ```json
  {
    "messageId": "msg-uuid-123",
    "rating": "LIKE", // or "DISLIKE"
    "feedback": "The explanation about taxes is incorrect." // comments (dislike comments)
  }
  ```

#### 2. Fetch Export Document Formats
- **Endpoint**: `GET v2/api/exports/formats`

#### 3. Export Bot Message
- **Endpoint**: `POST v2/api/messages/{messageId}/exports`
- **Payload**: `{ "format": "pdf", "fileName": "document.pdf" }`
- **Response**: Returns document file binaries (Binary Blob) with `content-disposition` header.

#### 4. Refresh Media URL Token
- **Endpoint**: `GET v2/api/task-media/{token}/refresh`

#### 5. MCP (Model Context Protocol) Connection Management
- `GET v2/api/mcp-connection/available`: Retrieves active MCP plugins supported on the platform.
- `POST v2/api/mcp-connection/{code}/connect`: Initiates connection oauth (returns authorization redirect Url).
- `DELETE v2/api/mcp-connection/{code}`: Tears down connection.
- `PATCH v2/api/mcp-connection/{code}/enabled`: Toggles active usage status.
- `PATCH v2/api/agent/{agentId}`: Patches agent parameters to enable/disable tools accessibility.

## 3. Streaming Chat Service (SSE Ask Protocol)

Ask requests use HTTP POST streaming via Server-Sent Events (SSE).

- **Endpoint**: `POST v2/api/messages/ask`
- **Payload**:
  ```json
  {
    "question": "User query text",
    "agentId": "agent-uuid-123",
    "conversationId": "conversation-uuid-456", // Optional (if continuing conversation)
    "attachmentIds": ["attachment-uuid-1"],    // Associated uploaded file references
    "enableWebSearch": true,                   // Enables search tool integration
    "timeZone": "Asia/Ho_Chi_Minh"             // Client's active timezone
  }
  ```

### SSE Stream Event Processing
Each line begins with the prefix `data: `. Upon receiving, slice this prefix off and parse the remaining text as JSON:

| SSE Line Payload | Type / Structure | Client Action / Side Effect |
| :--- | :--- | :--- |
| `data: [START]` | String | Marks initial payload ingestion start. |
| `{ "type": "conversation", "id": "...", "name": "..." }` | Object (Conversation Metadata) | Emitted on new chats. Client saves `conversationId` and redirects routing path to detail URL. |
| `{ "messageId": "...", "responseTime": 0.5 }` | Object | Attaches the active bot message database ID and response elapsed metric. |
| `{ "reasoning": "Processing information..." }` | Object (Reasoning stream) | Fragment tokens representing thoughts. Accumulated in client's thoughts accordion. |
| `{ "type": "tool_status", "tool": "web_search", "label": "Searching Google..." }` | Object (Tool execution) | Signals third-party tool execution. Client displays active loader status in chat stream. |
| `{ "response": "Welcome! " }` | Object (Content stream) | Markdown-formatted string fragments. Pushed into `tokenQueue` to render smoothly. |
| `{ "references": [...] }` | Object (Citations) | Citations/sources matching response content. |
| `{ "error": "Internal Error", "statusCode": 500 }` | Object (Error) | Fired when server errors mid-way. Client sets status to `'error'` and shows alert block with Retry trigger. |
| `data: [DONE]` | String | Signals connection close. Client sets `isStreaming = false` and terminates animation loops. |
